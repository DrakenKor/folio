## 4. IMPLEMENTATION NOTES

### Image Sourcing Strategy
1. **Commission a photographer:** Japanese craftspeople emphasize process and authenticity. Consider hiring a photographer familiar with craft documentation (similar to the aesthetic of Japanese craft magazines like Mingei or Kasane).

2. **Alternatively, reach out to makers directly:** Many artisans have their own documentation. Licensing existing high-quality studio photography may be faster than commissioning.

3. **Maintain consistency:** All images should:
   - Use natural daylight only (north light preferred)
   - Avoid excessive shadows or harsh contrast
   - Use muted, warm color grading
   - Feature minimal but contextual styling
   - Show clear material/craft details

### Color Grading & Post-Processing
- Warm color temperature (3500-4500K)
- Slightly lifted blacks (avoid pure black)
- Muted saturation—no oversaturated colors
- Subtle warm tint across shadows
- Maintain detail in highlights
- Consistent across all images for visual coherence

### Technical Requirements
- High resolution originals (at least 2x the display size)
- RGB color space for web
- Optimized exports for web (WebP format preferred with JPEG fallback)
- Consistent aspect ratios as specified
- Metadata stripped for privacy

### File Organization

### Update Locations

All images use the `img()` helper function which generates URLs from seeds. Update the seeds in code or replace the function to serve local files.

#### Current `img()` function (can be replaced with local file paths):
- **products.ts** (lines 4-5): `const img = (seed: string, width: number, height: number) => https://picsum.photos/seed/${seed}/${width}/${height}`
- **makers.ts** (lines 3-4): Same function definition

#### Option 1: Keep function, update image storage
1. Host images at a CDN or local `/public` directory
2. Modify `img()` function to construct proper paths: `const img = (seed: string, width: number, height: number) => /images/${seed}-${width}x${height}.jpg`

#### Option 2: Direct file path replacement
1. Replace each `img('seed-name', width, height)` call with actual file path
2. Update files directly (see checklist below)
