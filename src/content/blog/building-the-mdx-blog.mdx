I was thinking of a blog system dynamic enough to accommodate enriched html with embeds, as well as a search function, working within the constraints of github pages (Static hosting). I settled on markdown-first '.mdx' format blog posts 
where the routes are generated at build time, with interpolated JSX.

## Keep the route surface static

The blog works because the dynamic parts are decided before deployment. Posts are discovered from `src/content/blog`, frontmatter is validated once, and the blog routes are generated ahead of time so the site still fits `output: 'export'`.

```ts
const post = getPostBySlug(slug)

<MDXRemote source={post.content} components={mdxComponents} />
```

## Tight use of JSX

I used MDX for a small, curated component surface instead of turning posts into unrestricted React files. That gave me headings, links, callouts, demos, and room for visual pieces like the doodle SVG, while the writing still feels like markdown instead of application code.

<Callout type="warning" title="March 31, 2026">
  I finished this at the end of Q1 while the axios supply-chain compromise was unfolding. Public reporting on March 30 and March 31, 2026 identified malicious npm releases `axios@1.14.1` and `axios@0.30.4`. This repo does not install axios, but the incident still made me feel like the defaults in this repo feel correct: exact versions, a lockfile, `min-release-age`, and `ignore-scripts`.
</Callout>

That was the shape I wanted: markdown for speed, JSX for the rare expressive parts, and a blog system small enough that I can still trust what it is doing.
